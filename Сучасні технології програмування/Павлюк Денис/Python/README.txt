Тема: Розробка інструменту для перетворення зображень у текстову графіку (ASCII-арт) мовою Python.
1. Опис програми: Програма є консольною утилітою, яка приймає на вхід графічний файл (JPEG, PNG тощо) і генерує його текстову репрезентацію. Зображення формується з друкованих символів, підібраних таким чином, щоб імітувати відтінки сірого оригінального фото. Результат виводиться на екран та зберігається у текстовий файл.
2. Технічна реалізація:
--Бібліотека Pillow: Використовується модуль Image для низькорівневої роботи із зображеннями (відкриття файлів, доступ до масиву пікселів, зміна розміру, конвертація колірних просторів).
--Об'єктно-орієнтоване програмування: Логіка інкапсульована в класі AsciiArtConverter. Це забезпечує модульність коду та можливість його розширення.
--File I/O: Реалізовано читання зображень та запис текстових даних з використанням контекстних менеджерів (with open(...)).
3. Алгоритм роботи: Процес конвертації складається з чотирьох етапів:
--Завантаження: Зчитування файлу з диска.
--Зміна розміру:
----Зображення зменшується до заданої ширини (наприклад, 100 символів).
----Оскільки символи у терміналі не квадратні (висота > ширини), висота зображення стискається з коефіцієнтом 0.55. Без цього кроку фінальна картинка виглядала б розтягнутою по вертикалі.
--Десатурація (Grayscale Conversion):
----Зображення переводиться в режим L (Luminance). Кольоровий піксель (R, G, B) перетворюється на одне число (0..255), що відображає його яскравість.
--Мапінг (Pixel to Char Mapping):
----Кожен піксель замінюється на символ із наперед визначеного масиву ASCII_CHARS.
----Темні пікселі (близькі до 0) замінюються "густими" символами (@, #).
----Світлі пікселі (близькі до 255) замінюються "легкими" символами (., пробіл).
----Формула індексу: index = pixel_brightness // (255 / count_of_chars).

Для перевірки роботи можна використати файл із зображенням собаки dog.jpg, який вкладено у дану папку.
