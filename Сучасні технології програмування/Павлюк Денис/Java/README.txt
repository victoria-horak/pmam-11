Тема: Реалізація системи керування розумним будинком з використанням принципів ООП та графічної бібліотеки Java Swing.
1. Опис роботи програми: Програма емулює центральний хаб розумного будинку. Користувач бачить список пристроїв (Лампи, TV, Термостати), кожен з яких має унікальну назву та тип. Через графічний інтерфейс можна вмикати/вимикати окремі пристрої або керувати всіма одночасно (сценарії "Увімкнути все", "Вимкнути все"). Стан пристроїв відображається візуально (зміна кольору, тексту).
2. Технічна реалізація:
--Інтерфейс Switchable: Це фундамент системи. Він зобов'язує всі пристрої мати методи turnOn() та turnOff(). Завдяки цьому контролер HomeController може зберігати всі пристрої в одному списку List<Switchable>, не знаючи, чи це лампа, чи телевізор.
--Абстрактний клас SmartDevice: Використовує механізм успадкування для реалізації спільного коду (зберігання імені name та статусу on), щоб не дублювати його в кожному класі пристрою.
--Поліморфізм: Це ключова концепція завдання. Коли ми викликаємо метод device.turnOn() у циклі, Java автоматично визначає, який саме об'єкт знаходиться у змінній (Light, TV чи Thermostat) і викликає відповідну реалізацію методу. Лампа просто засвітиться, а термостат почне гріти — хоча команда одна й та сама.
3. Графічний інтерфейс (Swing):
--JFrame & JPanel: Використовуються для побудови вікна та сітки елементів.
--Event Dispatch Thread (EDT): Метод SwingUtilities.invokeLater гарантує, що інтерфейс запускається в окремому потоці, що запобігає зависанню програми.
--Динамічний UI: Метод refreshUI() повністю перемальовує список пристроїв при кожній зміні стану, забезпечуючи актуальність відображення даних.
